# Motiva — 動機フレームワーク

このプロジェクトではMotivaが有効です。以下のルールに従ってください。

## 自動動作

- セッション開始時にゴールとギャップが自動分析されます
- 各ツール使用前に不可逆アクション（git push, rm -rf 等）がチェックされます
- 停滞が検知されると自動的にリカバリー提案が行われます

## コマンド

- `motive status` — 現在の状態確認
- `motive goals` — アクティブなゴール一覧
- `motive add-goal --title "..." --description "..." --type deadline|dissatisfaction|opportunity` — ゴール追加
- `motive log` — アクションログ確認
- `motive reset` — セッション状態のリセット（ゴールは保持）

## コンテキスト

`.claude/rules/motive.md` に自動生成される動機コンテキストを参考にしてタスク優先度を判断してください。
このファイルはセッション開始時に更新され、現在のゴール・ギャップ・信頼残高が反映されます。

## タスク優先度の判断基準

1. `motive.md` に記載されたアクティブゴールの優先スコアを確認する
2. スコアが高いゴールに関連するタスクを優先して取り組む
3. ギャップが大きいゴールほど緊急度が高い

## 重要なルール

- **信頼残高が低い場合**（trust < 0.5）は、より慎重に行動し、不確実な操作の前にユーザーに確認を求めてください
- **不可逆アクションがブロックされた場合**は、必ずユーザーに確認を求めてください。ブロックを回避しようとしてはいけません
- **停滞リカバリー提案に従って**、同じアプローチを繰り返すのではなく、別の方法を試みてください
- **ゴールが未定義の場合**は、作業開始前にユーザーにゴールを確認することを推奨します
